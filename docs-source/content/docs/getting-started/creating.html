{{#pageData}} { "name": "Creating a Sate site" } {{/pageData}} {{ The first Sate command you will run is create. Sate create establishes all the scaffolding and boilerplate necessary to drive a Sate website. Pass the create command either an empty directory, or the name of a directory you want Sate to create.

{{
~sate-cms create ./my-sate-website

After create does its thing, cd into the new directory and check out the structure of your new Sate website. 

~cd ./my-sate-website


my-sate-websitels

The directory structure you'll see is something like this: 

    ./
        website.json
        index.html 
        about.html
        /styles
        /sate-cms
            /error
            /tempmlates
            /plugins
        /images

We'll go deeper into this structure in the next section. For now, try out this website by using the develop command: 

my-sate-websitesate-cms develop

You'll see a log of console output telling you that Sate is compiling the site. Then you'll see Sate starting the development webserver, and listening on port 3000. If you need to change the default port, use the -p flag with the develop command.

Now point your browser at http://localhost:3000 (or whatever port you're using), and you'll see the Sate boilerplate website.
